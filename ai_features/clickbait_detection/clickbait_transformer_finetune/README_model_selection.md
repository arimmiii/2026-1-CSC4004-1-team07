# Clickbait Model Selection Notes

이 문서는 클릭베이트 탐지 모델 선택만 따로 정리한 메모입니다.

## 현재 Transformer 결과

- 현재 메모의 수치는 이전 `concat_titles`/fast 실험 기록입니다.
- `title/body` split 재학습 후 갱신 예정입니다.
- 모델: `klue/roberta-base`
- 학습 설정:
  - `train subset 120,000`
  - `valid subset 20,000`
  - `test subset 20,000`
  - `epochs=1`
  - `max_length=96`
- 빠른 validation 결과:
  - Macro F1: `0.8241`
  - Weighted F1: `0.8243`
  - Accuracy: `0.8243`
- 저장된 fast 모델의 전체 test 결과 (`58,675` samples):
  - Macro F1: `0.8194`
  - Weighted F1: `0.8194`
  - Accuracy: `0.8194`

## 해석

- fast validation 결과와 전체 test 결과 차이가 크지 않음
- 즉, 현재 Transformer fast 모델은 과하게 튄 결과는 아님
- 다만 아직 `subset + 1 epoch` 결과라 최종 전체학습 점수는 아님

## 전체학습을 해볼 가치가 있는가

- 있음
- 이유:
  - 현재가 전체 데이터 학습이 아님
  - `epochs=1`, `max_length=96`의 빠른 실험임
  - 전체 `469,392` train + `max_length=128` + `2 epoch` 정도로 가면 소폭 상승 가능성이 있음

## 최종 모델 선택 기준

현재 서비스는 클릭베이트 탐지 외에도 아래 기능이 추가될 예정임:

- 정치 편향도 분석
- 뉴스 팩트체크
- 회원가입 및 일반 서비스 기능

이 조건에서는 클릭베이트 탐지 모델 하나가 너무 무거우면 전체 서비스 운영비와 응답속도에 불리함.

### Linear SVM을 유지하는 편이 나은 경우

- Transformer 전체학습 결과가 `Macro F1 0.82대 초반` 정도에 머무를 때
- SVM 대비 개선폭이 `0.01` 미만일 때
- CPU 서버 위주로 가볍게 운영해야 할 때
- 전체 서비스 기능이 많아서 추론 비용을 아껴야 할 때

이 경우에는:

- SVM이 더 가볍고 빠름
- 서버비가 적게 듦
- 배포와 유지보수가 단순함
- 다른 기능에 자원을 더 배분할 수 있음

### Transformer를 채택할 만한 경우

- 전체학습 후 `Macro F1 0.83 후반 ~ 0.84+` 정도로 올라갈 때
- SVM 대비 개선폭이 `0.015 ~ 0.02` 이상일 때
- 클릭베이트 탐지 정확도가 서비스 품질에 중요한 핵심 지표일 때

이 정도면 모델 무게와 운영비를 감수할 이유가 생김.

## 현재 시점 권장 판단

- 지금 결과만 보면 Transformer는 가능성은 있음
- 하지만 아직 SVM을 확실히 이긴다고 말할 단계는 아님
- 따라서:
  1. Transformer 전체학습 1회 수행
  2. 최종 test 점수 확인
  3. `0.84` 근처까지 못 올라오면 SVM 채택 검토

## 발표/보고서에 쓰기 좋은 결론 형태

- Transformer도 실험했으며, fast 모델 기준 전체 test에서 `Macro F1 0.8194`를 확인함
- 전체학습 시 추가 향상 가능성은 있으나, 서비스 전체 기능과 운영 비용을 고려하면 성능 개선폭이 작을 경우 Linear SVM이 더 실용적임
- 최종 선택은 정확도뿐 아니라 추론 비용, 응답속도, 배포 복잡도를 함께 고려해야 함
